# Карта объяснения главы 36. Как пользоваться учебником

## Вопрос главы

Как завершить первый полный проход учебника так, чтобы читатель получил рабочую навигацию, а проект не сделал вид, будто черновик 36 глав уже является готовой книгой?

## Переход из главы 35

Глава 35 научила читать исследования аккуратно:

```text
утверждение -> дизайн -> результат -> эффект -> ограничения -> мост -> практика
```

Теперь эту же дисциплину нужно применить к самому учебнику.

Учебник тоже нельзя читать как:

```text
глава -> прием -> делаю
```

Нужно читать как:

```text
сигнал -> уровень вопроса -> маршрут -> понятия -> действие -> обратная связь -> граница
```

## Линия объяснения

### 1. Зачем нужна финальная навигация

Большой учебник легко превращается в склад сильных глав.

Если читатель не понимает, как входить в книгу, он будет искать знакомые слова: мотивация, дофамин, прокрастинация, выгорание, ИИ, лидерство. Тогда он может начать с середины и пропустить понятия, без которых глава будет понята как очередной набор советов.

Финальная глава должна дать карту использования.

### 2. Сначала показать основной порядок

Учебник имеет линейную лестницу:

```text
потеря контекста
-> внешний контур
-> мотивация
-> мозг и тело
-> обучение и преодоление
-> продуктивность и восстановление
-> ИИ и лидерство
-> практика и границы
```

Этот порядок нужен для первого чтения.

### 3. Потом показать чтение по сигналу

После первого знакомства учебник можно использовать как справочник.

Но вход должен быть не по любимому совету, а по сигналу:

- не могу начать;
- потерял контекст;
- нет мотивации;
- устал и не восстанавливаюсь;
- ИИ стал думать вместо меня;
- команда расползается по срочностям;
- сотрудник потерял включенность;
- нужно понять исследование;
- нужно собрать личную систему.

### 4. Ввести вопрос выбора уровня

Главный вопрос:

```text
какой уровень сейчас активен?
```

Возможные уровни:

- задача и контекст;
- мотивация и управляемость;
- тело, стресс, восстановление;
- обучение и преодоление;
- ИИ и инструменты;
- команда и лидерство;
- границы и доказательность.

### 5. Дать маршруты

Маршруты должны быть практическими, но не рецептурными.

Каждый маршрут включает:

- где начать;
- какие главы подключить;
- какой первый ход сделать;
- какую обратную связь смотреть;
- какую границу не пропустить.

### 6. Маршрут разработчика

Смысл маршрута: войти в туманную задачу, не потерять контекст, не отдать мышление ИИ.

Главы:

- 1, 4-6;
- 21;
- 26-27;
- 31-33;
- 35.

### 7. Маршрут тимлида

Смысл маршрута: видеть команду как среду действия.

Главы:

- 7-11;
- 20-25;
- 28-30;
- 31;
- 34-35.

### 8. Маршрут человека в перегрузе

Смысл маршрута: не лечить перегруз новыми протоколами продуктивности.

Главы:

- 11;
- 15;
- 20-25;
- 31-34.

### 9. Маршрут обучения

Смысл маршрута: строить способность понимать и делать трудное.

Главы:

- 3;
- 16-19;
- 26-27;
- 35.

### 10. Маршрут работы с ИИ

Смысл маршрута: использовать ИИ как усилитель, а не обход.

Главы:

- 5-6;
- 16;
- 19;
- 26-27;
- 31-35.

### 11. Маршрут личного контура

Смысл маршрута: собрать повторяемую систему работы без бюрократии и самоизноса.

Главы:

- 4-6;
- 20-22;
- 31-32;
- 34-35.

### 12. Как возвращаться к книге

Дать короткий protocol:

```text
1. Назови сигнал.
2. Определи уровень.
3. Выбери маршрут.
4. Прочитай одну-две опорные главы.
5. Сделай первый проверяемый ход.
6. Зафиксируй обратную связь.
7. Проверь границы.
```

### 13. Честно обозначить статус проекта

Финал должен сказать:

```text
первый полный черновик 36 глав создан;
следующая работа - ревизия
```

Ревизия включает:

- полноту;
- язык;
- источники;
- визуальный слой;
- связки;
- пробелы;
- примеры;
- повторения;
- приватность рабочих материалов.

## Что глава не должна делать

- Не повторять все главы.
- Не писать "теперь вы знаете все".
- Не делать мотивационное завершение.
- Не обещать, что маршруты решают состояние.
- Не прятать медицинские, организационные и доказательные границы.
- Не объявлять цель полностью завершенной.

## Проверка полноты

Глава полна, если после нее читатель может:

- выбрать линейный или ситуационный режим чтения;
- найти маршрут по своей задаче;
- понять, какие главы читать сначала;
- не перепутать маршрут с рецептом;
- применить первый ход с обратной связью;
- проверить границы;
- использовать учебник как справочник;
- увидеть, что следующий шаг проекта - ревизия, а не самодовольная точка.
