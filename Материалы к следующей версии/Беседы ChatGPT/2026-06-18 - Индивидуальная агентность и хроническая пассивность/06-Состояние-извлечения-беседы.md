## Что не удалось извлечь

Не удалось получить скрытые или редактированные research-артефакты, если они недоступны в публичном backend JSON shared-беседы. В файл `01-Извлеченная-беседа.md` намеренно не включены служебные элементы `system`, `tool`, `thoughts`, `code`, browser/tool displays, скрытые custom-instruction placeholders и технические заглушки вместо tool/research-выводов.

Сырой список ссылок из `safe_urls` содержит мусорные или промежуточные элементы: `localhost`, `utm_source=chatgpt.com`, citation-обрывки, Wikipedia/популярные страницы. Для книги его нельзя использовать как готовую библиографию без проверки по DOI, PubMed/PMC, издательским страницам или официальным документам.

## Предыдущая неудачная попытка через локальный прокси 8877

2026-06-18 проверен локальный прокси `http://127.0.0.1:8877`.

Результат:

- `CONNECT chatgpt.com:443` через этот прокси вернул `502 Bad Gateway`;
- `CONNECT example.com:443` и `CONNECT google.com:443` тоже вернули `502`;
- `CONNECT claude.ai:443` и `CONNECT anthropic.com:443` прошли до внешних сайтов;
- порт `8877` слушает Python-процесс `claude-local-helper-proxy.py`;
- в коде helper'а явно задан allowlist только для `claude.ai`, `claude.com`, `anthropic.com`, `claudeusercontent.com`.

Вывод: этот локальный прокси технически работает, но предназначен для Claude/Anthropic и не пропускает `chatgpt.com`. Для извлечения ChatGPT shared-ссылок нужен отдельный общий HTTPS-прокси или временный helper с разрешенными доменами `chatgpt.com`, `openai.com` и связанными CDN/API-доменами.

## Успешное извлечение через новый общий прокси 8899

Новый общий прокси работает как цепочка:

```text
локальное приложение
-> http://127.0.0.1:8899
-> HTTPS proxy proxy-ru-nl.dalamar81.ru:443 с Basic auth
-> ru-dc
-> nl-hh
-> Internet
```

Проверка пользователя показала, что туннель работает:

- `curl.exe -x http://127.0.0.1:8899 https://ifconfig.me` вернул внешний адрес `80.66.84.151`;
- `curl.exe -x http://127.0.0.1:8899 -I https://chatgpt.com/` дошел до Cloudflare и получил `403 Forbidden` с `Cf-Mitigated: challenge`.

`403` на главную страницу `chatgpt.com` не означает поломку прокси: это Cloudflare-challenge на browser-facing endpoint. Backend JSON удалось получить отдельным запросом с браузерным `User-Agent`, `Accept: application/json` и `Referer` на shared-ссылку:

```powershell
curl.exe -x http://127.0.0.1:8899 -L --max-time 90 --compressed `
  -o "$env:TEMP\chatgpt-share-6a3400a0-6054-83eb-aa89-f6db1cb0f99c.json" `
  -A "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/125.0 Safari/537.36" `
  -H "Accept: application/json,text/plain,*/*" `
  -H "Referer: https://chatgpt.com/share/6a3400a0-6054-83eb-aa89-f6db1cb0f99c" `
  "https://chatgpt.com/backend-api/share/6a3400a0-6054-83eb-aa89-f6db1cb0f99c"
```

Этот способ следует считать рабочим рецептом для похожих shared-ссылок, если прямой доступ к `chatgpt.com` из текущей сети снова упирается в Cloudflare или таймауты.

## Надежный рабочий порядок для похожих ссылок

1. Сначала пробовать `https://chatgpt.com/backend-api/share/<id>` с браузерными заголовками и `Referer` на исходную shared-ссылку. Если прямой запрос блокируется, повторить через `http://127.0.0.1:8899`.
2. Если backend API доступен, извлекать `title`, цепочку от `current_node` до корня через `mapping`, видимые `message.content.parts` ролей `user` и `assistant`, `metadata.content_references`, `safe_urls`.
3. Служебные элементы JSON не смешивать с материалом для книги: `system`, `tool`, `thoughts`, `code`, displays, hidden custom instructions, placeholders и redacted tool outputs.
4. Если backend API недоступен, пробовать браузерный рендер shared-страницы и сохранять видимый текст.
5. Если share не показывает research-результаты, просить пользователя приложить их отдельно или экспортировать текст ответа из ChatGPT.
6. В dossier обязательно разделять извлеченную беседу, приложенные research-результаты, источники/DOI, редакторскую выжимку и протокол ограничений извлечения.
7. Не считать ChatGPT citation-маркеры вида `turn...` полноценными ссылками на источники. Для книги использовать DOI, PubMed, publisher pages, официальные guidelines и авторские страницы, указанные в research-тексте или найденные отдельной проверкой.

## Риск для будущего агента

Будущий агент может считать этот dossier рабочей копией видимой части shared-беседы и приложенного research-пакета. Но он не должен считать сырой `safe_urls` готовой библиографией и не должен предполагать, что публичная shared-ссылка содержит все research-артефакты без отдельной пользовательской выгрузки.
