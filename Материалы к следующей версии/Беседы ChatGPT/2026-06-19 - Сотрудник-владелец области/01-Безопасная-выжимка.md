# Безопасная выжимка

## Что извлечено

Из чувствительной беседы сохранена только общая тема, полезная для книги: сотрудник, который закрывает область целиком, принципиально ценнее сотрудника, которого нужно непрерывно контролировать.

Исходный личный контекст, чувствительные детали, частные обстоятельства и сама shared-ссылка не переносятся.

## Центральное различение

Исполнитель задачи действует по схеме:

```text
получил задачу -> уточнил -> сделал -> отдал -> ждет следующую
```

Владелец области действует по другой схеме:

```text
понимает цель области
держит ее текущее состояние
видит риски
сам замечает необходимость действия
принимает локальные решения
эскалирует только существенное
оставляет внешний след
обновляет модель области после обратной связи
```

Первый сотрудник часто возвращает руководителю скрытую работу: удержание цели, контекста, ограничений, рисков, приоритетов и критериев качества. Второй сотрудник забирает часть этого контура на себя.

## Почему постоянный контроль дорог

Сотрудник, требующий непрерывного контроля, может быть старательным и технически сильным, но при этом постоянно расходовать чужую рабочую память.

Типичный слабый запрос:

```text
Что делать?
```

Сильный запрос:

```text
Ситуация такая.
Факты такие.
Вижу три варианта.
Я бы выбрал второй, потому что он снижает главный риск.
Есть один эффект за пределами моих полномочий, поэтому хочу свериться.
```

Во втором случае руководитель получает интерфейс к мышлению сотрудника, а не сырую передачу когнитивной нагрузки.

## Ценность владельца области

Владелец области:

- снижает управленческую нагрузку;
- уменьшает число восстановлений контекста;
- делает состояние области наблюдаемым;
- быстрее обнаруживает риски;
- приносит руководителю не хаос, а срез ситуации;
- улучшает систему, а не только закрывает поручения;
- делает команду устойчивее к отсутствию одного центрального носителя контекста.

## Главная формула

```text
исполнитель закрывает действие;
владелец области закрывает контур.
```

Контур включает наблюдение, интерпретацию, решение, действие, обратную связь, исправление последствий и обновление модели.
