# Материалы к следующей версии

## Назначение

Эта папка собирает входящий корпус для следующей версии книги: рецензии, обратную связь, замечания по текущему тексту, новые темы, новые источники и спорные места.

Материалы здесь не являются текущей версией книги. Это накопитель для будущего большого редакторского прохода, когда агент в режиме Цели будет использовать весь корпус, чтобы подготовить новую версию учебника.

## Версионные якоря

Все ссылки на номера глав внутри этого каталога относятся к snapshot `CV1` - первой публичной/current reader edition текущей книги. Номера глав полезны как быстрые указатели, но в следующей версии они могут измениться.

Стабильная карта `номер -> название -> файл -> поисковый якорь` лежит в `02-Стабильные-якоря-текущей-версии.md`. Результат текущей проверки локальной целостности лежит в `03-Аудит-организации-и-ссылочной-целостности.md`. При переносе материала в новую версию нужно искать не "ту же главу по номеру", а смысловой адрес: stable ID `CV1-CHxx`, название главы, ключевые понятия и контекст материала.

## Как добавлять материалы

- Сохранять каждый самостоятельный материал отдельным Markdown-файлом.
- По возможности начинать имя файла с даты в формате `YYYY-MM-DD`.
- Если материал относится к конкретной главе, указывать номер, название главы и по возможности stable ID из `02-Стабильные-якоря-текущей-версии.md`, например `CV1-CH21 / глава 21 / Фокус, WIP и переключения`.
- Все существенные материалы заносить в `01-Реестр-материалов.md`.
- Если материал содержит только сырую мысль, все равно сохранять ее, но помечать как `разобрать`.

## Каталоги

- `Рецензии/` - внешние и внутренние рецензии на текущую версию книги.
- `Обратная связь/` - письма, комментарии, заметки читателей и фрагменты обсуждений.
- `Правки по главам/` - замечания, привязанные к конкретным главам или частям книги.
- `Новые темы/` - темы, которые стоит добавить или существенно расширить в следующей версии.
- `Новые источники/` - статьи, книги, исследования, ссылки и библиографические кандидаты.
- `Сомнения и спорные места/` - места, где есть риск ошибки, чрезмерного упрощения, слабой доказательности или конфликтующих интерпретаций.
- `Беседы ChatGPT/` - извлеченные shared-беседы, оформленные как карточки, беседы, источники и заметки для книги.

## Для будущего агента

Перед подготовкой новой версии книги нужно:

1. Прочитать `README.md`, этот навигатор, `02-Стабильные-якоря-текущей-версии.md`, `01-Архитектура-и-оглавление.md`, `02-Карта-понятий-и-пробелов.md`, `05-Реестр-глав.md` и текущие главы.
2. Прочитать все материалы из этой папки и сверить их с `01-Реестр-материалов.md`.
3. Разделить входящий корпус на правки текста, структурные изменения, новые темы, источниковые уточнения и спорные места.
4. Для каждой ссылки на номер главы сначала определить stable ID `CV1-CHxx` и смысловой адрес, а уже потом решать, в какую будущую главу или раздел переносить материал.
5. После внесения правок обновить связанные реестры, источники, проверки и экспортные артефакты.
