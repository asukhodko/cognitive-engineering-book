# Стабильные якоря текущей версии

## Назначение

Этот файл фиксирует, к какой версии книги относятся ссылки на номера глав в материалах для следующей версии. Главы в будущей редакции могут быть переупорядочены, объединены или разделены, поэтому номер главы нельзя считать стабильным адресом. Номер остается полезным быстрым указателем, но должен читаться вместе с ID snapshot, названием главы, путем файла и поисковыми словами.

## Как читать ссылки на главы

- `7` значит `CV1-CH07`, а не "седьмая глава любой будущей версии".
- `4-7` значит диапазон `CV1-CH04`...`CV1-CH07`.
- `1-36`, `вся книга`, `архитектура`, `источники`, `экспорт` являются широкими указателями на весь snapshot или на служебные слои, а не на будущую нумерацию.
- Если будущая версия перенумерует главы, сначала искать по stable ID ниже, затем по названию, пути файла, смысловому ядру и связанным понятиям.
- Новые материалы желательно помечать и номером, и stable ID: например, `CV1-CH21 / глава 21 / Фокус, WIP и переключения`.

## Правило для будущей версии

При переносе материалов в новую версию не делать механическую замену "глава 21 -> новая глава 21". Для каждого материала сначала найти смысловой адрес в `CV1`, затем решить, куда этот смысл переехал в новой структуре. Если глава разделена, материал может лечь в несколько новых мест; если главы объединены, несколько старых ID могут указывать на один новый раздел.
