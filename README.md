# Когнитивное инженерство

Рабочий репозиторий учебника по когнитивному инженерству: главы, паспорта глав, карты объяснения, источниковые пакеты, проверки, утилиты сборки и готовые экспортные артефакты.

Учебник объясняет проектирование мышления, действия, среды, восстановления и работы с инструментами. Он идет от проблемы потери состояния мысли и входа в задачу к мотивации, управляемости, цене усилия, нейрофизиологии, обучению, прокрастинации, выгоранию, ИИ как внешнему когнитивному контуру и лидерству как проектированию среды действия.

## Структура

- `Главы/` - 36 reader-facing глав учебника.
- `Паспорта/` - паспорта глав: цель, границы, учебные элементы и критерии.
- `Карты объяснения/` - карты понятий и переходов для глав.
- `Источники/` - библиографические реестры, source packages и evidence policy.
- `Проверки/` - audit trail редакторских, источниковых и структурных проверок.
- `Материалы к следующей версии/` - накопитель рецензий, обратной связи, новых тем, источников и спорных мест для следующего большого редакторского прохода.
- `Инструменты/` - сборка reader edition, EPUB, проверки и стили EPUB.
- `Экспорт/` - собранные Markdown и EPUB-артефакты.

## Материалы к следующей версии

`Материалы к следующей версии/` предназначены для накопления всего, что должно быть рассмотрено при подготовке следующей версии книги. Это входящий корпус, а не текущий текст учебника.

- `00-Навигатор.md` - назначение папки, правила добавления материалов и инструкция для будущего агента.
- `01-Реестр-материалов.md` - единый список входящих материалов, статусов, приоритетов и решений.
- `Рецензии/` - внешние и внутренние рецензии на текущую версию.
- `Обратная связь/` - комментарии читателей, письма, заметки и обсуждения.
- `Правки по главам/` - замечания, привязанные к конкретным главам.
- `Новые темы/` - темы для добавления или существенного расширения.
- `Новые источники/` - новые исследования, книги, статьи и библиографические кандидаты.
- `Сомнения и спорные места/` - места с риском ошибки, слабой доказательности или конфликтующих интерпретаций.
- `Беседы ChatGPT/` - извлеченные shared-беседы, оформленные как карточки, беседы, источники и заметки для книги.

## Готовые артефакты

- `Экспорт/Когнитивное инженерство - reader edition.md` - единая Markdown-версия для чтения.
- `Экспорт/Когнитивное инженерство - reader edition draft.md` - legacy draft-copy того же reader edition.
- `Экспорт/epub/Когнитивное инженерство.epub` - EPUB-версия книги.
- `Экспорт/epub/Когнитивное инженерство - epub source.md` - Markdown-источник для EPUB после преобразования Mermaid-диаграмм в изображения.
- `Экспорт/epub/assets/diagrams/` - Mermaid-исходники и PNG-рендеры диаграмм.

## Быстрый старт

```bash
make check
```

Пересобрать Markdown reader edition:

```bash
make reader
```

Пересобрать EPUB:

```bash
make epub
```

Проверить только библиографию:

```bash
make bibliography
```

## Зависимости

Минимум для Markdown-проверок:

- Python 3.

Для EPUB-сборки:

- `pandoc`;
- Java и `epubcheck`;
- Mermaid CLI (`mmdc` из `@mermaid-js/mermaid-cli`);
- Chrome/Chromium для рендера Mermaid через Puppeteer.

Если `mmdc` или Chrome находятся не в стандартных местах, можно задать:

```bash
export MMDC_BIN=/path/to/mmdc
export PUPPETEER_EXECUTABLE_PATH=/path/to/chrome-or-chromium
```

## Основные проверки

```bash
python3 "Инструменты/check_reader_edition.py"
python3 "Инструменты/check_goal_evidence.py"
python3 "Инструменты/check_bibliography.py"
python3 "Инструменты/check_epub_export.py"
java -jar /usr/share/java/epubcheck.jar "Экспорт/epub/Когнитивное инженерство.epub"
```

Текущая baseline-сборка:

- 36 глав;
- reader edition: `1748769` bytes;
- EPUB: 54 диаграммы;
- библиография: 302 записи, без DOI/URL/title/author-year дублей и без author-year gaps.

## Границы

Учебник не является медицинским справочником, психотерапевтическим протоколом, HR-инструкцией или набором универсальных техник продуктивности. Он проектирует условия действия и явно отмечает клинические, организационные, evidence-level и fast-moving границы.
