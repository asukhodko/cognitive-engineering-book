# Пакет источников для главы 20

## Статус

Дата проверки: `2026-05-25`.

Глава: [[../Главы/20-Продуктивность-без-самоизноса]].

Назначение: источниковая опора для главы о продуктивности как устойчивом режиме работы, а не как выжимании ресурса. Глава должна открыть часть VI и перевести читателя от опыта преодоления к проектированию условий, в которых трудное действие можно повторять без накопления разрушительного долга.

Ключевая редакционная задача: не сделать еще одну главу с советами "как успевать больше". Нужно объяснить, что продуктивность в когнитивном инженерстве держится на трех условиях:

```text
ресурсный пол
-> защищенный вход
-> рабочий поток с видимым сдвигом
-> завершение или checkpoint
-> восстановление
-> следующий вход дешевле
```

Противоположность этой петле:

```text
давление
-> больше часов и больше WIP
-> хуже восстановление
-> дороже вход
-> ниже качество
-> больше тревоги
-> еще больше давления
```

## Что этот пакет дает главе

Глава должна заменить простую формулу:

```text
продуктивность = больше часов + больше давления + больше задач
```

на инженерную:

```text
продуктивность = условия,
в которых ценное действие регулярно входит в рабочий поток,
оставляет проверяемый сдвиг
и не уничтожает будущую доступность действия
```

Для этого нужны четыре опоры:

1. Рабочая нагрузка должна иметь восстановление, иначе усилие превращается в долг.
2. День лучше оценивать не только по результатам, а по управляемым признакам режима.
3. Переключения и размытые паузы повышают цену следующего входа.
4. Выгорание нельзя лечить теми же приемами, которыми настраивают обычный рабочий ритм.

## Обязательные различения

| Понятие | Что важно не перепутать |
| --- | --- |
| Продуктивность | Не количество занятости, а устойчивое продвижение по ценным задачам с сохранением будущего входа. |
| Самоизнос | Не "трудная работа", а режим, где результат покупается разрушением восстановления, качества и будущей управляемости. |
| Ресурсный пол | Не батарейка счастья, а минимальное состояние системы, при котором действие остается доступным. |
| Хороший ритм | Повторяемая связка входа, фокуса, сдвига, выхода и восстановления. |
| Режим нажима | Увеличение часов, WIP и внутреннего давления при ухудшении восстановления и качества. |
| Рабочая пауза | Пауза, которая возвращает в тот же рабочий контекст; не равна личному отдыху и не равна прокрастинационному облегчению. |
| Объяснимость времени | Диагностика режима; не тайм-трекинг ради контроля и самообвинения. |
| DoD дня | Управляемые признаки качества рабочего дня; не только внешние KPI и не только список завершенных задач. |

## Что не делать в главе

- Не писать "работай меньше" как универсальный совет. Речь о режиме, а не о простом сокращении часов.
- Не подменять продуктивность отдыхом. Восстановление нужно, чтобы действие оставалось доступным.
- Не подменять восстановление развлечением: некоторые развлечения открывают новый контекст и повышают цену возвращения.
- Не использовать burnout как общее слово для любой усталости.
- Не обещать, что правильный ритуал, трекер или тайм-блокинг решит организационную перегрузку.
- Не давать медицинские рекомендации по лечению выгорания, депрессии, хронического недосыпа или тревожных состояний.
- Не делать из главы список личных привычек автора; внутренние заметки нужно переводить в общие инженерные принципы.
