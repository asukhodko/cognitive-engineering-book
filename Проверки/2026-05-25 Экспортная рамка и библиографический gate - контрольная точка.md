# Экспортная рамка и библиографический gate - контрольная точка

Дата проверки: `2026-05-25`.

## Что сделано

Созданы два новых служебных артефакта:

- [[../07-Публичный-front-matter-и-экспортная-рамка]] - публичный вход, описание книги, аудитория, ограничения, правила экспорта и минимальная аннотация.
- [[../Источники/02-Финальная-библиография-и-export-gate]] - граница между source-hygiene для `ready-for-review` и будущей публикационной библиографией для `done`.

## Зачем это нужно

После закрытия финальной языковой редактуры всех блоков 1-36 учебник стал цельным reader-facing корпусом, но оставался риск смешать книгу с production-аппаратом vault.

Новая экспортная рамка фиксирует:

- что читательская книга не должна экспортировать паспорта, карты объяснения, проверки, source packets и сырой research-пакет;
- что текущий корпус имеет статус `ready-for-review`, не `done`;
- что production-хвосты глав нужно либо очищать, либо выносить в единую библиографию;
- что быстро меняющиеся источники требуют отдельной проверки перед публикацией.

## Что сознательно не делалось

- Не составлялась полная публикационная библиография: текущие source packets достаточны для `ready-for-review`, но не заменяют финальный список литературы.
- Не переводились главы в `done`.
- Не удалялись production-хвосты из всех глав: для этого нужен отдельный экспортный проход.
- Не обновлялись через интернет fast-moving источники по ИИ, software engineering и новой нейронауке.

## Текущее решение

Для будущего публичного экспорта выбран основной режим:

```text
Reader edition = главы 1-36 + публичный front matter + финальная библиография
```

Production-слои остаются внутри vault:

```text
00-06, паспорта, карты объяснения, проверки, source packets, raw research package
```

Источниковые хвосты глав в reader edition лучше не оставлять как есть. Их нужно либо:

- очистить от локальных ссылок и служебных статусов;
- либо заменить единой финальной библиографией.

## Следующий редакционный фронт

Следующий крупный фронт перед `done`:

1. Собрать единую финальную библиографию.
2. Обновить fast-moving источники на дату публикации.
3. Провести экспортную санитаризацию глав 1-36.
4. После этого принимать решение по статусам глав.
