# Связка глав 4-6

Дата проверки: `2026-05-24`.

Проверяемый блок:

```text
Глава 4. Контекст задачи: что нужно вынести из головы
-> Глава 5. Рабочий журнал как внешний контур мышления
-> Глава 6. Ритуалы входа и выхода
```

## Главный вопрос проверки

Блок должен быть не набором трех близких глав, а учебной лестницей:

```text
состав контекста -> ведение состояния во времени -> повторяемый цикл входа и выхода
```

Если главы повторяют друг друга, блок нужно сжать. Если между ними есть скачок, нужно добавить мост. Если в конце нет практического инструмента, блок не выполняет функцию первой прикладной части учебника.

## Результат

Связка работает как завершенный первый практический блок.

После главы 4 читатель знает, какие элементы состояния задачи нужно вынести наружу.

После главы 5 читатель понимает, как эти элементы живут во времени через рабочий журнал.

После главы 6 читатель получает повторяемую процедуру:

```text
открыть журнал -> восстановить состояние -> назвать туман -> выбрать первый проверяемый шаг -> выполнить блок -> оставить checkpoint
```
