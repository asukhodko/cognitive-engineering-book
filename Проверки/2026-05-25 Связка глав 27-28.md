# Связка глав 27-28

Дата: `2026-05-25`.

Связка: [[../Главы/27-Как-работать-с-ИИ-не-отдавая-ему-субъектность]] -> [[../Главы/28-Лидерство-как-дизайн-среды-действия]].

## Что должна сделать связка

Глава 27 показывает, что при работе с ИИ субъектность сохраняется через контур:

```text
цель
-> роль инструмента
-> проверка
-> решение
-> след
```

Глава 28 должна перенести эту логику с инструмента на командную среду:

```text
цель
-> постановка задачи
-> границы автономии
-> WIP
-> обратная связь
-> решение команды
-> устойчивый результат
```

| Есть ли мост к командному WIP и выгоранию? | Да. Глава задает основу для главы 30: WIP, прерывания, перегруз и восстановимость режима. |

## Что глава 28 добавляет

Новые понятия:

- среда действия команды;
- лидерство как дизайн условий;
- задача как мотивационный интерфейс;
- командный внешний контур;
- каденции как петли обратной связи;
- автономия в границах;
- безопасное сообщение о проблеме;
- командная управляемость;
- метрики как приборная панель.

Главное новое различение:

```text
лидерство не включает мотивацию напрямую;
лидерство меняет среду,
в которой мотивация, фокус,
управляемость и обучение становятся
более или менее доступными
```

## Что проверить при будущей ревизии

1. Не нужно ли глубже проверить и интегрировать primary sources по psychological safety.
2. Не слишком ли глава опирается на внутренние TL-материалы; финальная версия должна быть универсальной.
3. Достаточно ли в главе практических примеров задач как интерфейса.
4. Не стоит ли часть блока про метрики вынести в будущие практические кейсы.
5. Не нужно ли добавить визуальную мини-карту "задача как интерфейс" в финальной редакции.
