# Связка глав 28-29

Дата: `2026-05-25`.

Связка: [[../Главы/28-Лидерство-как-дизайн-среды-действия]] -> [[../Главы/29-Мотивация-сотрудников]].

## Что должна сделать связка

Глава 28 показывает лидерство как дизайн командной среды:

```text
цель
-> постановка задачи
-> границы автономии
-> WIP
-> обратная связь
-> устойчивый результат
```

Глава 29 должна приблизить эту рамку к конкретному человеку:

```text
человек
-> задача
-> среда
-> состояние
-> обратная связь
-> опыт действия
-> мотивационный прогноз
```

## Проверка перехода

| Вопрос | Ответ |
| --- | --- |
| Есть ли явный мост из главы 28? | Да. Глава 29 начинает с того, что лидер не включает мотивацию напрямую, а работает с условиями, в которых мотивация становится доступной. |
| Не повторяет ли глава 28? | Нет. Глава 28 говорит о среде команды, глава 29 - о мотивации конкретного сотрудника внутри задачи и среды. |
| Не стала ли глава манипулятивной? | Нет. В тексте прямо разведены мотивация и манипуляция, мотиватор и ярлык, честная связь задачи с ценностью и "продажа" плохой среды. |
| Разведены ли индивидуальный и системный уровни? | Да. Глава показывает, когда помогает индивидуальный разговор, а когда проблема находится в WIP, требованиях, справедливости, полномочиях или восстановлении. |
| Есть ли визуальная опора? | Да. Есть схема мотивационной петли сотрудника и диагностическая таблица потери мотивации. |
| Не забирает ли глава 30? | Частично вводит WIP, перегруз и командную среду только как границу; подробный разбор командного фокуса, прерываний и выгорания оставлен главе 30. |
| Санитаризированы ли внутренние материалы? | Да. TL Matrix используется как структура тем без имен, внутренних кейсов, ссылок, HR-sensitive деталей и рабочих обстоятельств. |

## Что глава 29 добавляет

Новые понятия:

- мотивация сотрудника как контекстное состояние;
- мотиватор как гипотеза, а не ярлык;
- мотивационный профиль задачи;
- гигиенический пол мотивации;
- разговор о мотивации без оценки личности;
- индивидуальная и системная работа с мотивацией;
- граница мотивационного нажима.

Главное новое различение:

```text
мотивация сотрудника - не кнопка и не черта;
это отношение между человеком,
задачей,
средой,
состоянием,
обратной связью
и накопленным опытом действия
```

## Риски, которые нужно удержать при ревизии

| Риск | Контроль |
| --- | --- |
| Превратить главу в HR-инструкцию. | Оставить фокус на когнитивно-инженерных переменных: ценность, автономия, компетентность, принадлежность, управляемость, цена усилия, обратная связь. |
| Превратить мотиваторы в типологию людей. | Удерживать формулу "мотиватор - гипотеза, не ярлык". |
| Переподать "продажу задачи" как полезную технику. | Оставить жесткую границу: продавать можно только честную связь, плохую среду нужно менять. |
| Смешать главу с burnout-главами. | Оставить burnout/boreout как диагностическую границу, не разворачивать повторно весь материал глав 23-25. |
| Смешать главу с главой 30. | Командный WIP и прерывания упоминать только как системные причины, а не подробно раскрывать. |
| Сделать сильный claim по Herzberg без проверки. | В источниковом пакете Herzberg line оставлена как `to-verify`; в главе используется осторожное различение гигиенического пола и мотивационного роста. |

## Проверка последовательности

Последовательность выдержана:

```text

30: командный WIP, прерывания и выгорание разбираются как системные эффекты среды
```

## Что проверить при будущей ревизии

1. Не нужно ли заменить термин "гигиенический пол" на более нейтральный, если он будет звучать слишком бытово.
2. Проверить primary sources по Herzberg перед финальной редакцией.
3. Добавить более сильный пример из инженерной задачи без приватной рабочей конкретики.
4. Проверить, не слишком ли много содержания главы 30 уже раскрыто в конце главы 29.
5. Проверить язык на скрытые менеджерские клише: "вовлеченность", "продать", "заряженность", "мотиватор".
