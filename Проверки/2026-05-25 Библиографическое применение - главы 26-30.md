## Evidence gate

| Блок | Evidence-роль | Редакционная граница |
| --- | --- | --- |
| ИИ как внешний когнитивный контур | `strong` для offloading/distributed cognition/automation risk; `fast-moving` для AI evidence | Не писать "ИИ ускоряет" или "ИИ разрушает мышление" как общий закон. |
| Субъектность при работе с ИИ | `strong` для trust calibration, retrieval and self-efficacy; `context-dependent` для режимов использования | Не превращать главу в набор промптов и не делать один режим универсальным. |
| Лидерство как дизайн среды | `strong` для SDT, JD-R, control/recovery; `context-dependent` для leadership transfer | Не переносить внутренние рабочие кейсы и не обещать всемогущество локального лидера. |
| Мотивация сотрудников | `strong` для SDT/self-efficacy/work design; `mixed` для Herzberg heuristic; `clinical-boundary` для тяжелой просадки | Не делать типологию сотрудников, HR-инструкцию или манипулятивную "продажу задачи". |
| Командный фокус и выгорание | `strong` для switching/interruption/recovery/JD-R/burnout boundary; `context-dependent` для командных договоров | Не превращать главу в agile-инструкцию, запрет прерываний or диагностику burnout. |
