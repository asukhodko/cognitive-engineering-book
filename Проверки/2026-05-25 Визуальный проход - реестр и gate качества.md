# Визуальный проход - реестр и gate качества

Дата проверки: `2026-05-25`.

## Что проверялось

Фронт из исходной цели учебника: материалы должны постоянно сопровождаться хорошими иллюстрациями и/или диаграммами.

Проверялись два уровня:

- наличие визуальной опоры в каждой главе;
- наличие рабочего gate, который не позволит считать главу готовой только потому, что в ней есть любая таблица или mermaid-схема.

## Автоматический срез

Проверка по главам показала:

```text
главы 1-36: в каждой есть хотя бы один mermaid-блок
главы 1-36: в каждой есть markdown-таблицы
```

Это подтверждает покрытие по наличию, но не доказывает качество визуального слоя.

## Редакционное решение

Визуальная готовность главы теперь делится на два статуса:

```text
present
-> схема или таблица есть

quality-pass
-> визуальная опора объясняет механизм, развилку, уровень доказательности или практический маршрут
```

Текущий корпус закрывает `present`, но не закрывает `quality-pass` для всех глав. Это значит, что статус `ready-for-review` остается корректным, а перевод в `done` без визуальной редакции запрещен.

## Gate перед `done`

Перед переводом главы в `done` нужно проверить:

1. какой вопрос читателя закрывает схема;
2. появляется ли схема до перегруза;
3. разобрана ли схема текстом после вставки;
4. не вводит ли схема новый термин молча;
5. не создает ли схема ложную точность;
6. помогает ли она выбрать действие, увидеть границу или удержать различение.

Если визуальная опора не проходит эту проверку, глава получает хвост `needs-visual-quality-pass`.
