# Проверка связки глав 22-23

Дата: `2026-05-25`.

Связка: [[../Главы/22-Ресурсность-сила-и-ритуалы]] -> [[../Главы/23-Как-ломается-мотивационный-контур]].

## Что должна сделать связка

Глава 22 показывает:

```text
ресурсность - это рабочий режим,
в котором вход, удержание и возврат
имеют приемлемую цену
```

Глава 23 должна объяснить следующий уровень:

```text
если ресурсность системно не возвращается,
нужно разбирать не отдельный ритуал,
а поломку мотивационного контура
```

Переход:

```text
ресурсный режим делает правила фокуса выполнимыми
-> при нормальном сбое помогают восстановление, ритуалы и WIP-правила
-> при системном сбое ломаются связи ценности, усилия, управляемости, обратной связи и восстановления
```

## Проверка перехода

| Вопрос | Ответ |
| --- | --- |
| Есть ли явный мост из главы 22? | Да. Глава 23 начинается с ситуации, где ритуалы и восстановление уже не возвращают доступность входа. |
| Не повторяет ли глава 22? | Нет. Глава 22 описывает ресурсный режим и его сборку; глава 23 описывает разрыв мотивационного контура, когда режим системно не возвращается. |
| Не забирает ли глава 24? | Нет. Глава 23 только обозначает два маршрута поломки; подробное различение burnout/boreout оставлено главе 24. |
| Не забирает ли глава 25? | Нет. Глава 23 дает диагностику места разрыва, но не полный протокол восстановления. |
| Есть ли новая центральная схема? | Да: нормальный мотивационный контур и контур поломки через требования, цену усилия, управляемость, угрозу, неприсвоенный результат и восстановительный долг. |
| Есть ли практический выход? | Да: таблица диагностики места разрыва и пять проверок перед мотивационным нажимом. |
| Есть ли границы? | Да. Глава явно не диагностирует burnout, депрессию, тревожные и хронические состояния и указывает границы самостоятельной работы. |

## Что глава 23 добавляет

Новые понятия:

- мотивационный контур;
- поломка мотивационного контура;
- предвыгорание;
- восстановительный долг;
- health-impairment process;
- high demand / low resource;
- effort-reward imbalance;
- потеря авторства результата;
- ментальная дистанция;
- цинизм как защитная экономия;
- редукция профессиональной эффективности;
- мотивационный нажим.

Главное новое различение:

```text
усталость - временный рост цены усилия,
поломка контура - системное расхождение ценности,
управляемости, цены, обратной связи и восстановления
```

## Риски, которые удалось удержать

| Риск | Как удержан |
| --- | --- |
| Использовать burnout как синоним усталости. | Введена граница ICD-11: burnout как профессиональный феномен хронического рабочего стресса. |
| Романтизировать предвыгорание. | Предвыгорание описано как зона, где вовлеченность может покупать результат восстановительным долгом. |
| Свести проблему к личной дисциплине. | Использована рамка требований, ресурсов, управляемости, обратной связи и восстановления. |
| Сделать цинизм моральным диагнозом. | Цинизм описан как возможная защитная дистанция, без оправдания плохого поведения. |
| Перейти к лечению. | Глава ограничена диагностикой контура и выводом к главам 24-25. |
| Смешать burnout и boreout. | В главе они только обозначены как два маршрута, подробное различение оставлено следующей главе. |

## Проверка последовательности

Последовательность выдержана:

```text
20: продуктивность сохраняет будущий вход
21: WIP и переключения меняют цену входа
22: ресурсный режим делает правила исполнимыми
23: если ресурсный режим не возвращается, ломается мотивационный контур
24: разные маршруты поломки - burnout и boreout
25: восстановление как возвращение управляемости
```

## Что проверить при ревизии

1. Не слишком ли глава 23 заранее раскрывает содержание главы 24.
2. Достаточно ли ясно различены усталость, недовосстановление и burnout.
3. Не звучит ли "авторство результата" слишком новым термином без достаточной опоры.
4. Не нужно ли добавить короткий инженерный пример из работы разработчика или тимлида без приватных деталей.
5. Не перегружена ли глава источниковыми рамками JD-R, demand-control, effort-reward и COR для одного читательского шага.
