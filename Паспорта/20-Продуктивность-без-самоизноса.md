# Паспорт главы 20. Продуктивность без самоизноса

## Задача главы

Открыть часть VI и отделить продуктивность от токсичной эффективности. Показать, что в когнитивном инженерстве продуктивность означает не максимальное выжимание часов и силы, а проектирование режима, в котором ценное трудное действие регулярно входит в рабочий поток, оставляет проверяемый сдвиг и не разрушает будущую доступность действия.

Глава должна перевести читателя от вывода главы 19:

```text
способность делать трудное растет через управляемую трудность
```

к новому вопросу:

```text
как организовать работу так,
чтобы способность делать трудное не превратилась в самоизнос
```

## Читательский вход

К этому месту читатель уже знает:

- что сложная работа требует восстановления состояния задачи;
- что мотивация зависит от ценности, угрозы, управляемости, цены усилия и состояния;
- что усталость меняет цену будущего действия;
- что стресс и аллостаз задают цену режима, а не только цену отдельного дня;
- что сон, паузы и восстановление поддерживают обучение и следующий вход;
- что прокрастинация может маскироваться под отдых;
- что преодоление полезно только тогда, когда трудность остается управляемой и обучающей.

## Новые понятия

- продуктивность без самоизноса;
- ресурсный пол;
- защищенный вход в рабочий поток;
- рабочий поток;
- видимый сдвиг;
- контрольная точка дня;
- объяснимость времени;
- DoD работы;
- управляемые признаки дня;
- запаздывающие сигналы;
- рабочая пауза;
- хороший ритм;
- режим нажима;
- долг восстановления.

## Главная мысль

Продуктивность - это не способность долго игнорировать усталость. Это способность регулярно создавать ценный сдвиг в задачах, сохраняя будущую способность возвращаться к ним.

Центральная петля главы:

```text
ресурсный пол
-> защищенный вход
-> рабочий поток
-> видимый сдвиг
-> контрольная точка
-> восстановление
-> следующий вход дешевле
```

Противоположная петля:

```text
давление
-> больше часов / больше WIP
-> хуже восстановление
-> дороже вход
-> ниже качество
-> больше тревоги
-> еще больше давления
```

## Обязательные различения

| Различение | Что удержать |
| --- | --- |
| Продуктивность / занятость | Занятость может быть шумом; продуктивность оставляет ценный сдвиг и следующий вход. |
| Продуктивность / самоизнос | Продуктивность сохраняет будущую доступность действия; самоизнос покупает результат ухудшением будущего входа. |
| Ресурсный пол / "заряд батарейки" | Ресурсный пол - минимальные условия доступности действия, а не настроение и не полный бак счастья. |
| Рабочая пауза / личный отдых | Рабочая пауза восстанавливает внутри рабочего режима; личный отдых может открывать другой контекст и требует отдельной границы. |
| Рабочая пауза / прокрастинационное облегчение | Пауза делает следующий вход доступнее; прокрастинационное облегчение часто делает его дороже. |
| Объяснимость времени / полицейский учет минут | Объяснимость нужна для диагностики режима, а не для самонаказания. |
| Управляемые признаки / запаздывающие сигналы | Управляемыми можно рулить в пределах дня; запаздывающие только подтверждают траекторию. |
| Хороший ритм / режим нажима | Ритм повторяем и восстановим; нажим увеличивает давление при падающей управляемости. |
| Восстановление / награда | Восстановление - часть работы системы, а не премия после "достаточной" продуктивности. |

## Обязательная визуальная опора

Главная схема:

```mermaid
flowchart LR
    FLOOR["Ресурсный пол"] --> ENTRY["Защищенный вход"]
    ENTRY --> FLOW["Рабочий поток"]
    FLOW --> SHIFT["Видимый сдвиг"]
    SHIFT --> CHECK["Контрольная точка"]
    CHECK --> RECOVERY["Восстановление"]
    RECOVERY --> NEXT["Следующий вход дешевле"]
    NEXT --> ENTRY
```

Контрастная схема:

```mermaid
flowchart LR
    PRESSURE["Давление"] --> MORE["Больше часов и WIP"]
    MORE --> LESSREC["Меньше восстановления"]
    LESSREC --> EXPENSIVE["Дороже вход"]
    EXPENSIVE --> QUALITY["Ниже качество"]
    QUALITY --> ANXIETY["Больше тревоги"]
    ANXIETY --> PRESSURE
```

Дополнительная таблица: DoD рабочего дня.

| Слой | Примеры | Как использовать |
| --- | --- | --- |
| Управляемые признаки | объяснимость времени, меньше распада на переключения, видимый сдвиг, граница работы и отдыха | Проверять в конце дня и ремонтировать режим завтра. |
| Запаздывающие сигналы | внешняя оценка, завершенные задачи, метрики, стабильное качество | Смотреть как подтверждение траектории, но не пытаться управлять ими напрямую сегодня. |

## Практический пример

Рабочий день разработчика, лида, автора или исследователя:

```text
утром есть ресурсный пол
-> первый блок защищен от чатов и новостей
-> задача открывается через внешний контекст
-> выбирается один видимый сдвиг
-> на выходе фиксируется контрольная точка
-> пауза не уводит в новый личный контекст
-> вечером день можно объяснить
```

Плохой вариант:

```text
поздний старт
-> чаты до первого фокуса
-> много мелких входов
-> паузы как развлечения
-> важная задача не получила сдвига
-> вечернее давление
-> сон и восстановление снова срезаны
```

## Популярные ошибки, которые глава должна предотвратить

- "Продуктивность - это делать больше дел".
- "Если я могу потерпеть, значит можно держать такой режим постоянно".
- "Отдых нужно заслужить".
- "Если день был тяжелый, надо просто сильнее контролировать себя завтра".
- "Пауза внутри работы может быть любым приятным занятием".
- "Если я не могу объяснить день, нужно лучше трекать минуты".
- "Внешние метрики важнее управляемых признаков режима".
- "Выгорание лечится новым расписанием, трекером или ритуалом".
- "Ресурсность - это настроение, мотивация или эйфория".

## Границы главы

Глава не является медицинским, психотерапевтическим или HR-протоколом. Она не диагностирует выгорание, депрессию, тревожные расстройства, хронический недосып или эндокринные нарушения.

Если у человека уже заметна траектория выгорания, нарушены сон, здоровье, отношения, безопасность или способность возвращаться после отдыха, productivity-приемы должны уступить место снижению нагрузки, восстановлению, внешней поддержке и профессиональной помощи.

Задача главы - дать инженерную модель устойчивого рабочего режима. Подробные главы про фокус, WIP, ресурсность, поломку мотивационного контура, burnout/boreout и восстановление идут дальше.
